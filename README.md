# Academic Personal Website

A GitHub Pages site that auto-builds from Google Drive data, using Hugo + LaTeX.

**Live demo:** https://manolo-assistant.github.io/academic-site/

## Architecture

```
Google Sheet (published CSV)
  ├── conferences tab ──→ Travel page
  ├── talks tab ────────→ Individual talk pages (with abstracts)
  ├── publications tab ─→ Publications list
  └── teaching tab ─────→ Teaching section
         │
         ▼  (daily cron + on push)
  GitHub Actions (.github/workflows/build.yml)
         │
         ├── scripts/sync-drive.py    → downloads CSV, generates Hugo content
         ├── scripts/build-cv.py      → generates PDF CV from same data
         └── hugo --minify            → builds static site
         │
         ▼
  GitHub Pages
```

## Data Source

Everything comes from one Google Sheet (except images, which go in a Drive folder).

### Current spreadsheet tabs:
- **conferences** (gid=0): Travel/events — `id, title, url, location, tags, date, date_end`
- **talks** (gid=1581332120): Talks with abstracts — `id, title, type, event, url, abstract, tags, date, date_end`

### Planned tabs (see SPREADSHEET-SCHEMA.md):
- **publications**: Papers with arXiv IDs, journal info
- **teaching**: Courses taught
- **bio**: Site-wide settings
- **positions**: Academic positions
- **education**: Degrees
- **awards**: Fellowships and awards

Until these tabs exist, the sync script uses hardcoded fallback data.

## Setup

### 1. Publish your Google Sheet
- File → Share → Publish to web → Each tab as CSV
- Note the published CSV URLs

### 2. Configure
Edit `scripts/sync-drive.py` to update the `DEFAULT_CSV_URLS` dict with your published URLs.

Or use `--sheet-id` to download tabs by name from any sheet (requires the sheet to be publicly readable).

### 3. Images
Create a public Google Drive folder for images (profile photo, etc.).
Pass `--image-folder-id FOLDER_ID` to sync-drive.py, or set `GOOGLE_DRIVE_FOLDER_ID` as a GitHub secret.

### 4. Deploy
Push to `main`. GitHub Actions will build and deploy.

Enable Pages: Settings → Pages → Source: **GitHub Actions**.

## Local Development

```bash
# Generate content from spreadsheet
python scripts/sync-drive.py

# Start Hugo dev server
hugo server
```

## Migrating to your own account

1. Fork/copy this repo to your GitHub account
2. Update `baseURL` in `hugo.toml`
3. Update published CSV URLs in `scripts/sync-drive.py`
4. Enable GitHub Pages
5. Point your custom domain (CNAME) to `<username>.github.io`

## File Structure

```
├── .github/workflows/build.yml   # CI/CD: daily cron + push
├── content/                       # Hugo content (generated by sync-drive.py)
│   ├── _index.md                  # Homepage / bio
│   ├── publications/              # Papers
│   ├── talks/                     # Individual talk pages with abstracts
│   ├── travel/                    # Conferences & travel list
│   └── teaching/                  # Courses
├── cv/
│   └── templates/cv.tex           # LaTeX CV template
├── data/cv.json                   # Structured data for LaTeX CV
├── layouts/                       # Custom Hugo templates
├── scripts/
│   ├── sync-drive.py              # Google Sheet → Hugo content
│   └── build-cv.py                # cv.json → LaTeX → PDF
├── static/
│   ├── css/style.css              # Stylesheet
│   └── cv.pdf                     # Auto-generated CV
├── SPREADSHEET-SCHEMA.md          # Planned unified spreadsheet schema
└── hugo.toml                      # Hugo config
```
